# 我搭建博客的心路历程

最开始写博客是在简书，优点是方便，缺点是少了点个性

后面在网上找教程尝试第一次用 Hexo 来搭建博客

  • 优点：Hexo 社区活跃，生态丰富，有很多漂亮的主题与功能强大的插件
  • 缺点：
    • bug 多：总是报各种奇怪的错误
    • 编译速度太慢了
    • 貌似不支持热更新： hexo server 不支持热更新，每次更改配置与内容后都得手动重启本地服务，这真的挺麻烦的

抛弃 Hexo，使用 Hugo

  • 优点：
    • 编译速度快：Hugo 基于 GO 语言，比起用 Node.js 编写的 Hexo 来说，编译速度快了许多
    • 支持热更新：启用 hugo server 后，不管是 .md 文件还是配置文件，只要存在改动，它都会自动刷新配置与内容（目前我只发现 hugo 支持配置热更新）
  • 缺点：
    • 主题太少了，对比 Hexo，好看的 Hugo 主题还是太少了

抛弃 Hugo，使用 VuePress

  • 优点：
    • 好看！其实 VuePress 默认的主题更像是文档，用了 vuepress-theme-reco 这个主题之后看起来才像是一个博客 其实是因为这个主题才开始用的 VuePress
    • VuePress 支持许多 Markdown 扩展语法，比如代码块中的行高亮
    • 页面切换速度快：因为是基于 Vue 的 SPA 应用
    • VuePress 内置搜索插件，不用自己折腾了
  • 缺点：
    • 只有部分支持热更新：貌似只有在写 .md 时才支持热更新，更改 config.js 和侧边栏都不支持热更新

抛弃 VuePress，回到 Hexo 的怀抱

本来我都打算常驻 VuePress 了，但奈何大佬 @霜月琉璃的主题 shaka 实在是太漂亮了；果然对于我来说，实用性什么的都只是暂时的，好看才是一辈子的事儿，呜呜呜。
